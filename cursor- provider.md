# API Endpoints & Capabilities Guide: usage App

This document details the capabilities, headers, payloads, and JSON response structures of the Cursor dashboard endpoints queried by the **"usage"** application.

---

## 1. Authentication Strategy
The endpoints authorize requests using standard session cookie authentication. 
*   **Cookie Name**: `WorkosCursorSessionToken`
*   **Format**: `<userId>::<jwt_token>` (where the `userId` matches the `sub` claim inside the decoded JWT).
*   **Extraction Method**: The **usage** app automatically extracts the active JWT from the local SQLite database at `~/Library/Application Support/Cursor/User/globalStorage/state.vscdb` (key: `cursorAuth/accessToken`).

## 2. API Endpoints

### Endpoint 2.1: Usage Summary (`GET`)
Retrieves the overall plan details, active billing cycle dates, and remaining quotas.

*   **URL**: `https://cursor.com/api/usage-summary`
*   **Method**: `GET`
*   **Headers**:
    ```http
    Cookie: WorkosCursorSessionToken=<userId>::<jwt>
    Origin: https://cursor.com
    Referer: https://cursor.com/dashboard/usage
    ```
*   **Response Structure (JSON)**:

### Endpoint 2.2: Filtered Usage Events (`POST`)
Fetches individual request events, including token splits, model types, request contexts, and calculated costs, over a specified Unix millisecond date range.

*   **URL**: `https://cursor.com/api/dashboard/get-filtered-usage-events`
*   **Method**: `POST`
*   **Headers**:
    ```http
    Content-Type: application/json
    Cookie: WorkosCursorSessionToken=<userId>::<jwt>
    Origin: https://cursor.com
    Referer: https://cursor.com/dashboard/usage
    ```
*   **Payload (JSON)**:
    ```json
    {
      "startDate": 1780826400000,
      "endDate": 1783515400000
    }
    ```
*   **Response Structure (JSON)**:
    ```json
    {
      "totalUsageEventsCount": 1420,
      "usageEventsDisplay": [
        {
          "timestamp": "2026-07-08T14:32:10.000Z",
    ```

## 3. Metrics Calculated by the **usage** App
Using the combination of these two endpoints, the **usage** app computes the following statistics in real-time:
*   **Projected Spend**: Cycles spend divided by elapsed time, projected to the end of the billing cycle.
*   **Token Splits**: Segment ratios for input (`In`), output (`Out`), cache write (`Cache W`), and cache read (`Cache R`) tokens.
*   **Average cost per request**: Sum of `chargedCents` divided by the total count of events.
*   **Background Activity Ratio**: Percentage of requests generated by background agents (`isHeadless: true`).
*   **Hourly Activity Grid**: A 24-hour block distribution mapping which hours of the day register requests.
